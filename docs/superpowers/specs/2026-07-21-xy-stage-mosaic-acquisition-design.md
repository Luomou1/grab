# XY 位移台实时拼图与空间扫描设计

## 目标

在现有相机、传感器 ROI 与 PZT 纵向扫描程序中加入 FMC01-02H + FMSXY100-20-20 双轴位移台，实现：

1. 按用户设置的 XY 空间范围进行概览扫描；
2. 扫描过程中实时生成可缩放的样品拼图；
3. 在拼图上框选空间 ROI；
4. 根据相机物理视场和重叠率自动生成覆盖网格与运动路径；
5. 位移台依次移动到每个 XY 瓦片位置，并在每个位置执行一次完整 PZT 纵向扫描；
6. 保存原始瓦片、纵向序列、概览拼图、坐标、标定、路径和恢复状态。

本设计以用户提供的初始像素间距 `0.48 µm/px` 建立默认标定，但实际运动前仍须通过 X/Y 多点移动确认轴方向、旋转、镜像和残余比例误差。

## 坐标模型

系统保留三个互不混用的坐标域：

- 传感器坐标：像素，控制相机输出 ROI；
- 样品坐标：mm，控制 XY 位移台和空间扫描范围；
- 纵向坐标：µm，控制现有 PZT 扫描。

相机像素与样品坐标使用二维仿射变换：

```text
[u, v]T = A * [X, Y]T + b
```

默认 `A` 的尺度由 `0.48 µm/px` 初始化。标定向导在安全中心位置分别执行 X/Y 多点移动，用相位相关估计图像平移并以最小二乘更新 `A`。标定指纹包含相机分辨率、传感器 ROI、旋转、镜像、位深和用户填写的光学配置名；任一关键项变化都使空间标定失效。

## 架构

### XY 位移台适配层

从独立位移台控制工程提取 SDK、模型、安全门禁和单一工作线程模式。采集程序不启动或控制外部 GUI 进程。所有厂家 DLL 调用只能发生在 XY 工作线程中。

适配层提供连接、断开、状态快照、回零、使能、绝对双轴移动、全轴停止和控制器软限位读取。自动扫描还要经过独立的应用安全范围校验，不能仅依赖可能未配置的控制器软限位。

### 空间扫描总控

新增唯一的 `SpatialScanWorker`，按状态机串行拥有相机、XY 位移台和 PZT 扫描任务：

```text
IDLE -> SURVEY -> SURVEY_READY -> ACQUIRING -> FINALIZING -> COMPLETED
                                   |                |
                                   +-> STOPPING ----+
                                   +-> FAILED
```

概览点的顺序为：移动、等待双轴到位、稳定、软触发新帧、复制图像、排队保存、提交拼图。空间采集点的顺序为：移动、到位、稳定、执行完整 PZT 纵向扫描、记录结果、进入下一点。

停止操作同时设置总控取消事件、停止 PZT、向 XY 工作线程提交高优先级全轴停止，并将相机恢复为连续触发模式。

### 拼图与实时地图

实时拼图采用“位置先验 + 图像修正”：

1. 根据标定矩阵和 XY 读回位置立即放置瓦片缩略图；
2. 后台仅在预计重叠区域使用相位相关计算平移修正；
3. 低置信度时保留位移台位置并标记警告；
4. 实时界面只维护低分辨率画布和覆盖遮罩；
5. 扫描结束后使用所有相邻约束做全局平移最小二乘优化；
6. 原始数据不做不可逆融合，显示拼图使用线性羽化。

界面新增“相机预览/样品地图”双页。样品地图显示当前瓦片、已采集瓦片、失败瓦片、位移台当前位置、用户 ROI 和规划路线。现有传感器 ROI 框选继续只作用于相机。

## 覆盖与路径规划

相机物理视场由当前帧尺寸和标定尺度计算。默认重叠率为 20%，允许配置 10%–40%。规划器生成完全覆盖用户 ROI 的最小网格，而不是固定四等分；当网格为 2×2 时即对应四份。

默认使用蛇形路径减少空程，并提供单向行扫描选项以降低步进位移台回程误差。所有瓦片中心和相机视场边界必须位于应用安全范围内。规划前输出瓦片数、预计 XY 路程、PZT 扫描次数和估计耗时。

## 数据与恢复

每个任务保存为独立目录：

```text
SpatialScan-时间戳/
  job.json
  calibration/
  survey/tiles/
  survey/tile_index.csv
  survey/preview.png
  plan/selected_roi.json
  plan/route.json
  acquisition/tile_rNNN_cNNN/
  summary.json
```

`job.json` 和 `route.json` 使用原子替换更新。每个瓦片完成后记录状态，因此中断后可跳过已完成瓦片。每张图记录 XY 目标/读回位置、PZT 目标/读回位置、帧号、触发与采集时间、曝光、增益、相机 ROI、拼接修正和相关置信度。

第一版继续使用当前 PNG/TIFF 原始图保存方式，并生成低分辨率 PNG 概览，避免本次强制引入大型图像格式依赖。数据模型为后续 OME-Zarr 或金字塔 OME-TIFF 导出保留接口。

## 故障与安全

- 连接后必须通过位移台参数指纹和软限位读取；
- 自动扫描要求双轴使能、空闲、无报警且位于应用安全范围；
- 同一 COM 口不得与 FMotion 同时打开；
- 连续三次通信失败、驱动报警、限位、到位超时或保存失败都会停止自动任务；
- 软件停止不能代替硬件急停；
- DPOS 是开环规划位置，图像配准用于修正拼图但不能掩盖机械丢步；
- 空白或周期纹理导致的低置信度配准不得覆盖位移台坐标先验；
- 定量原始数据与显示融合结果严格分离。

## 测试与验收

### 无硬件测试

- 仿射坐标正反变换和 `0.48 µm/px` 初始尺度；
- 覆盖网格、蛇形/单向路径、边界与软限位；
- 相位相关平移、低置信度回退和实时画布更新；
- XY 模拟控制器连接、到位、超时、限位和取消；
- 总控概览扫描、空间扫描、保存失败、停止和断点状态；
- 主界面构建、地图框选和控件锁定。

### 硬件验收

- 0.1–1 mm 小范围移动确认 X/Y 方向；
- 多点标定残差与往返误差；
- 2×2、3×3 概览扫描和拼图；
- 框选 2×2 区域后依次完成四次 PZT 纵向扫描；
- 扫描中停止、限位、相机超时、串口断开和磁盘写入失败；
- 长任务保存吞吐、内存占用和重新启动后的恢复信息。

验收条件是：无硬件测试全部通过；硬件小范围测试中无越界和设备并发访问；每个 XY 瓦片只对应一次完整 PZT 扫描；停止指令能同时终止 XY 与 PZT；原始数据、路径和坐标日志完整可追溯。
